请严格按照 Git Commit Message 规范生成提交信息，整体分为 Header、Body、Footer 三部分，段落之间必须用**空行**分隔。

一、整体格式要求
完整格式：
<类型>[可选作用域]: <简短描述>

[可选正文内容]

[可选脚注内容]

二、字段详细规则
1. Header（标题行，必填，50字符以内）
由 类型 + 作用域(可选) + 简短描述 组成：
(1) 类型type（必选，固定取值）
- init：代码初始化
- feat：新增功能
- fix：修复Bug
- refactor：代码重构（无新增功能、无Bug修复）
- docs：仅修改文档内容
- style：代码格式调整（空格、缩进、分号等，不改变代码逻辑）
- test：新增/修改测试代码
- chore：构建、辅助工具类修改，不改动src和test代码
- build：修改构建流程、依赖库、打包工具
- perf：代码性能优化
- ci：修改持续集成CI配置
- revert：版本回滚
(2) 作用域scope（可选，写在()内）
标注影响模块/范围，如global、common、http、数据库、*（代表全局多模块）；无明确范围可省略。
(3) subject简短描述：不超过50字符，简洁说明本次修改目的。

2. Body（正文，可选，重大修改必填）
- 每行建议72字符以内，可多行书写
- 第一字母小写，动词原形开头（第一人称现在时），结尾不加句号
- 必须说明三点：修改原因、解决方式、是否存在副作用/风险

3. Footer（脚注，可选）
- 存在破坏性变更：标注 BREAKING CHANGE，说明接口/版本/功能等不兼容改动
- 关联问题：填写关联Issue/任务ID、Bug链接，格式为 Closed + 问题ID

三、额外规则
1. 小幅度修改可省略 Body 和 Footer，仅保留 Header；
2. 输出结果只返回标准 Commit 内容，不要额外解释、多余文字；
3. 参考示例：
fix(common): 修复字体过小BUG，统一页面默认字体为14px

根据页面展示问题，全局调整通用模块字体配置，将所有页面默认字体修改为14px，无功能副作用。

Closed #123
